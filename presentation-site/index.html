Як запустити та протестувати Cardly

Ви можете встановити готовий мобільний додаток або розгорнути проєкт локально для розробки та ознайомлення з кодом.

  • Швидкий тест: Завантажте APK-файл за кнопкою вище та встановіть його прямо на свій Android-пристрій.
  • Локальний запуск: Клонуйте репозиторій за допомогою Git, перейдіть у папку проєкту та виконайте команди: npm install, а потім npm start.
  • Команда для збірки додатка, яка доступна в проєкті: npm run build:android:download.
Проєкт:

Cardly — інтерактивне навчання за допомогою карток із системою інтервальних повторень (SRS) та публічними колодами.

Технологічний стек:

Expo, React Native, Supabase, TypeScript

Рекомендоване використання:

Тестування, демонстрація та дистрибуція для рецензентів.

Шлях до APK:

presentation-site/app.apk

iOS / Android / Web

Усі три платформи працюють як один універсальний клієнт, створений на єдиній кодовій базі за допомогою Expo Router та React Native.

Процес навчання

Керування колодами, проходження квізів та розумне інтервальне повторення в одному плавному та інтуїтивному інтерфейсі.

Спільнота та приватність

Браузер публічних колод, інструменти скарг, а також гнучке налаштування видимості колод (публічні або приватні) та спільної роботи.

Ключові можливості

  • Створення колод: Створення нових колод для організації пов'язаних карток.
  • Додавання карток: Створення пар запитання/відповідь із текстом, зображеннями або аудіо.
  • Редагування карток/колод: Оновлення вмісту, зміна порядку карток або метаданих колоди.
  • Імпорт колод: Завантаження файлів CSV або зразків для масового додавання карток.
  • Навчання / Квіз: Запуск навчальних сесій із настроюваними режимами тестування.
  • Інтервальні повторення (SRS): Автоматичне планування карток для повторення на основі алгоритмів SRS.
  • Проходження квізів: Практика вивчення колод в ігровій формі.
  • Оцінювання відповідей: Позначення якості згадування для коригування графіка повторень.
  • Відстеження статистики: Перегляд прогресу навчання, серій днів (streaks) та ефективності кожної колоди.
  • Пошук та фільтрація: Пошук карток за текстом, тегами або статусом; фільтрація списків навчання.
  • Поширення / Експорт: Експорт колод або поширення посилань для спільної роботи.
  • Синхронізація / Беккап: Синхронізація прогресу між пристроями або експорт резервних копій.
  • Налаштування: Конфігурація теми, мови, нагадувань у сповіщеннях та навчальних преференцій.
  • Скарги / Прапорці: Позначення проблемних карток або надсилання відгуків для підтримки якості.

Структура проєкту

Мета: Cardly є кросплатформним додатком для навчання (Expo + web) із чіткою структурою: код додатка, асети, збірки під платформи, перевикористовувані пакети та невеликий сайт-презентація.

Верхній рівень (Root)

Містить конфігурацію додатка та файли збірки (app.json, eas.json, metro.config, package.json), а також папки платформ для Android та нативних інструментів.

Вихідний код (App source)

Головний додаток знаходиться в src/app — сторінки, екрани, роутинг та лейаути. UI побудований на компонентах, де велика папка components містить елементи на кшталт форм, модалок та базових UI-примітивів.

Стан та логіка

Контексти та хуки згруповані в src/contexts та src/hooks, що дозволяє тримати авторизацію, теми, мову та налаштування навчання централізованими й легкими для перевикористання.

Спільні дані та типи

Папка data/ містить початкові або демонстраційні колоди/картки; types/ та lib/ зберігають спільні типи TypeScript та хелпери для забезпечення узгодженості функцій.

Асети (Assets)

В assets/ зберігаються шрифти та зображення, які використовуються додатком та сайтом презентації, відокремлюючи медіафайли від коду.

Нативний Android

Директорія android/ містить конфігурації Gradle та нативні налаштування для Android, необхідні для збірки автономних додатків.

Пакети Monorepo

Папка packages/ містить ізольовані пакети (наприклад, srs/) для логіки інтервальних повторень або інших спільних бібліотек, що полегшує тестування.

Скрипти та інструменти

Папка scripts/ містить скрипти автоматизації (наприклад, синхронізація, хелпери інструментарію), які використовуються під час розробки та деплою.

Сайт-презентація

Невеликий статичний сайт знаходиться в presentation-site для маркетингу або документації; він незалежний від додатка, але використовує спільні асети.

Тестування та доки

Тести (де вони є) розміщені поруч із компонентами; документація та приклади даних для імпорту живуть у docs/ та docs/import-samples/.

Як орієнтуватися в проєкті

Почніть із точки входу додатка (лейаути/сторінки в папці app), потім вивчіть components для UI, contexts для стану, packages для основних алгоритмів та assets для медіа. Збірка та специфічні конфігурації платформ знаходяться в колі репозиторію та в android/.

Чому цей проєкт особливий

Cardly поєднує в собі повноцінний функціонал для створення флеш-карток із витонченим, адаптивним інтерфейсом та кросплатформеною доступністю. Проєкт демонструє практичне застосування сучасних технологій мобільної та веб-розробки, синхронізацію даних через хмару та орієнтований на користувача дизайн.
